# الدرس 5: المنطق والتحكم بمسار البيانات

> **الـ Workflow الجاهز**: [درس 5 - المنطق والبيانات](https://almearaj1ai.app.n8n.cloud/workflow/vPCgB8TYVHvUQyZJ)

لحد الحين الـ workflows حقنا تمشي بخط مستقيم. بالواقع تحتاج **قرارات**: إذا كذا سوّ هذا، وإلا سوّ ذاك.

## الـ Workflow الجاهز: تقرير الطقس الذكي

```
[Manual Trigger] → [HTTP Request: طقس الكويت] → [Set: استخراج الحرارة] → [IF: هل الجو حار؟]
                                                                              ├── true → [رسالة الجو الحار ☀️]
                                                                              └── false → [رسالة الجو اللطيف 🌤️]
```

هذا الـ workflow يجيب **طقس الكويت الحقيقي** من خدمة Open-Meteo المجانية (بدون مفتاح API)، يستخرج درجة الحرارة، وبعدين **يقرر** أي رسالة يطلع حسب الحرارة.

## نفّذه وشوف

1. افتح الـ workflow واضغط **Execute workflow**
2. لاحظ إن **مسار واحد بس** اشتغل (أخضر) والثاني ما اشتغل — هذا هو التفريع!
3. اضغط على نود "هل الجو حار؟ - IF" وشوف: كم item طلع من مخرج true وكم من false؟

## نود IF — القرار الثنائي

نود **IF** يقارن قيمة بشرط ويوجه كل item لأحد مخرجين:
- **true**: الشرط تحقق
- **false**: الشرط ما تحقق

في مثالنا الشرط: `temperature` أكبر من (gt) `35`

أنواع المقارنات حسب نوع البيانات:
- **String**: يساوي / يحتوي / يبدأ بـ...
- **Number**: أكبر / أصغر / يساوي... (gt, lt, equals)
- **Boolean**: صح / خطأ
- **Array**: فاضية / تحتوي / طولها...

## نود Switch — أكثر من مسارين

إذا تحتاج 3 مسارات أو أكثر (مثلاً: بارد / معتدل / حار / حار جداً) استخدم نود **Switch** — تكتب قاعدة لكل مخرج، وفي مخرج احتياطي (Fallback) لما ما تنطبق ولا قاعدة.

## نود Filter — تصفية بدون تفريع

**Filter** يشبه IF لكن بمخرج واحد: يمرر الـ items اللي تحقق الشرط ويرمي الباقي. ممتاز لتنظيف البيانات قبل معالجتها.

## نود Merge — دمج المسارات

عكس التفريع: ياخذ من مدخلين أو أكثر ويدمجهم. مفيد لما تجيب بيانات من مصدرين وتبي تكملهم كمسار واحد.

## Loop (تكرار) — معالجة عنصر عنصر

نود **Loop Over Items (Split in Batches)** يعالج البيانات على دفعات — مفيد لما عندك 500 عنصر وتبي تعالجهم وحدة وحدة أو 10-10 عشان ما تضغط على API خارجي.

## تمارين الدرس

1. **غيّر حد الحرارة**: افتح نود IF وغيّر الـ 35 إلى 20 ونفّذ — أي مسار اشتغل الحين؟
2. **أضف شرط ثاني**: في نفس نود IF أضف شرط إن `windSpeed` أكبر من 30 ولاحظ خيار **AND / OR** فوق الشروط — شنو الفرق بينهم؟
3. **تحدي**: استبدل نود IF بنود **Switch** بثلاث قواعد: أقل من 20 "بارد"، من 20 إلى 35 "معتدل"، أكثر من 35 "حار". (احفظ نسخة من الـ workflow قبل: قائمة ⋯ → Duplicate)

## شنو تعلمت؟

- ✅ HTTP Request لجلب بيانات حقيقية من API
- ✅ IF للتفريع الثنائي و Switch للتفريع المتعدد
- ✅ Filter و Merge و Loop

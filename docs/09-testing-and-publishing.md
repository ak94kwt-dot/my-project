# الدرس 9: الاختبار والنشر ومعالجة الأخطاء

آخر درس: كيف تخلي الـ workflow حقك **يشتغل بروحه** بشكل موثوق.

## Draft مقابل Active (المسودة والنشر)

كل الـ workflows اللي بنيناها لحد الحين **مسودات (Draft)** — تشتغل بس لما تنفذها يدوياً.

عشان الـ workflow يشتغل بروحه (Schedule يضرب بوقته، Webhook يستقبل طلبات، Chat يكون متاح للناس) لازم تنشره:

- افتح الـ workflow → زر **Active** (فوق يمين) → فعّله ✅

> 💡 **متى تنشر؟** بعد ما تتأكد إن كل شي يشتغل صح بالتنفيذ اليدوي. المسودة للتطوير، والنشر للإنتاج.

## سجل التنفيذات Executions — أداتك الأولى للتصحيح

من القائمة الجانبية → **Executions** تشوف كل تنفيذ صار:
- ✅ أخضر: نجح
- ❌ أحمر: فشل — اضغط عليه وبيفتح لك الـ workflow بحالته وقت الفشل، والنود اللي فشل عليه عليه علامة حمراء مع **رسالة الخطأ كاملة**

**طريقة قراءة أي خطأ**: اضغط على النود الأحمر → اقرأ الرسالة → غالباً وحدة من هذي:
- بيانات ناقصة (الحقل اللي تقرأه مو موجود) → شيك على التعبير والـ INPUT
- credential خطأ أو منتهي → جدد المفتاح
- الـ API الخارجي رفض الطلب → شيك على الـ URL والمعاملات

## التعامل مع الأخطاء داخل الـ Workflow

### إعدادات النود (Settings tab داخل أي نود)
- **Retry On Fail**: أعد المحاولة تلقائياً إذا فشل (ممتاز لأخطاء الشبكة المؤقتة)
- **On Error**: شنو يصير إذا فشل نهائياً؟
  - Stop Workflow (الافتراضي): يوقف كل شي
  - Continue: يكمل وكأن شي ما صار
  - Continue (using error output): يطلع الخطأ من مخرج خاص تعالجه ببرانش منفصل

### Error Workflow — شبكة الأمان
تقدر تسوي workflow خاص يشتغل **كل ما فشل أي workflow ثاني** (مثلاً يرسل لك تنبيه تيليجرام بالخطأ):
1. سوّ workflow جديد يبدأ بنود **Error Trigger**
2. في أي workflow ثاني: Settings ← **Error Workflow** ← اختره

## نصائح الإنتاج (Best Practices)

1. **سمّ النودات بأسماء واضحة** — "جلب طقس الكويت" مو "HTTP Request 1" (لاحظ كل دروسنا كذا)
2. **استخدم Sticky Notes** للتوثيق داخل الـ canvas — مثل الملاحظات الملونة في workflows الدروس
3. **جرب بالـ Manual Trigger أول** وبعدين بدّله بالـ trigger الحقيقي
4. **انسخ قبل التعديل الكبير**: قائمة ⋯ → Duplicate
5. **راقب الـ Executions** أول أيام بعد النشر

## تمارين الدرس الأخيرة 🎓

1. افتح **Executions** وشوف سجل كل تنفيذاتك من بداية الدورة.
2. اكسر شي عمداً: في workflow درس 5 غيّر الـ URL إلى رابط غلط ونفّذ — اقرأ رسالة الخطأ وافهمها، وبعدين رجّع الرابط الصح.
3. فعّل **Retry On Fail** على نود "جلب طقس الكويت" (Settings داخل النود).
4. **التخرج**: انشر (Active) workflow درس 7، وافتح رابط الشات وشاركه مع صديق يجرب ايجنتك! 🎉

# الدرس 1: ما هو n8n؟ المفاهيم الأساسية

## شنو هو n8n؟

n8n (تنطق "إن-إيت-إن" — اختصار nodemation) هي منصة **أتمتة** تخليك تربط خدمات وتطبيقات مع بعض وتسوي مهام تلقائية **بدون ما تكتب كود** (أو بكود بسيط إذا تبي).

أمثلة على أشياء تقدر تسويها:
- كل ما يوصلك إيميل مهم → يرسل لك تنبيه على تيليجرام
- كل يوم الساعة 8 الصبح → يجيب أسعار العملات ويحطها في جدول
- شخص يعبي فورم في موقعك → يضيفه على قاعدة بيانات ويرسل له إيميل ترحيب
- **ايجنت ذكاء اصطناعي** يرد على أسئلة العملاء ويستخدم أدوات بنفسه (هذا هدفنا في الدورة!)

## المصطلحات الخمسة الأساسية

هذي أهم 5 كلمات لازم تحفظها — كل شي في n8n مبني عليها:

### 1. Workflow (سير العمل)
الصفحة الكاملة اللي تبني فيها الأتمتة. مثل "الوصفة" — تحدد الخطوات من البداية للنهاية.

### 2. Node (نود / عقدة)
كل صندوق داخل الـ workflow. كل نود يسوي مهمة وحدة محددة: يجيب بيانات، يعدلها، يرسلها... إلخ. الـ workflow هو سلسلة نودات متصلة ببعض.

### 3. Trigger (مشغّل)
أول نود في أي workflow — هو اللي يقرر **متى** يبدأ التنفيذ:
- **Manual Trigger**: يشتغل لما تضغط زر (للتجربة والتعلم)
- **Schedule Trigger**: يشتغل بوقت محدد (كل ساعة، كل يوم...)
- **Webhook**: يشتغل لما يوصل طلب من برنامج خارجي
- **Chat Trigger**: يشتغل لما أحد يرسل رسالة في المحادثة (نستخدمه مع الايجنت)

### 4. Execution (تنفيذ)
كل مرة يشتغل فيها الـ workflow تنحفظ كـ "execution" — تقدر ترجع لها وتشوف شنو صار بالضبط في كل نود. هذي أهم أداة للتعلم وتصحيح الأخطاء.

### 5. Items (عناصر البيانات)
البيانات اللي تمشي بين النودات. كل نود يستقبل items من اللي قبله، يعالجها، ويطلعها للي بعده. كل item هو كائن JSON (بنشرحه في درس 3).

## شكل الـ Workflow

```
[Trigger] ──→ [نود 1] ──→ [نود 2] ──→ [نود 3]
  البداية      معالجة       معالجة       النتيجة
```

البيانات تمشي من اليسار لليمين، من نود إلى نود، عبر الخطوط اللي تربطهم.

## تعرف على واجهة n8n

افتح حسابك [almearaj1ai.app.n8n.cloud](https://almearaj1ai.app.n8n.cloud) وشوف:

1. **Overview / القائمة الرئيسية**: تشوف فيها كل الـ workflows حقك — المفروض تلقى 4 جاهزة (دروس 2، 5، 7، 8)
2. **Canvas / اللوحة**: لما تفتح workflow — هني تضيف نودات وتربطها
3. **زر + أو Add node**: يضيف نود جديد
4. **Executions**: سجل كل التنفيذات السابقة
5. **Credentials**: مفاتيح الربط مع الخدمات الخارجية (درس 6)

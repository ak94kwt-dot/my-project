# الديوانية — حزمة المنتج (v4.6)

## شنو هذا؟
محرّك توقّع ردة الفعل + موقع كامل. محاكي قواعدي يثبت التجربة قبل ربط النموذج اللغوي.

## بنية المشروع

| الملف | شنو يسوي |
|------|----------|
| `engine.js` | **المحرّك** — ٢٤ شخصية + رادار ٦ أبعاد + منطق الحكم. القلب. |
| `index-full.html` | **الموقع الكامل** — صفحة هبوط + تطبيق. افتحه للتجربة الكاملة. |
| `app.js` | **منطق الموقع** — يبني كل الصفحة ديناميكياً من JS. |
| `index.html` | نسخة مبسّطة (التطبيق فقط، بلا صفحة هبوط). |
| `llm-bridge.js` | **جسر النموذج اللغوي** — برومبت + قالب استدعاء جاهز للإنتاج. |
| `docs/engine-spec-v4.6.md` | الوثيقة الكاملة (الموت/الـMoat) — القواعد الذهبية، قاموس اللهجات، حلقة التعلّم الذاتي. |
| `docs/gap-report.md` | تقرير الثغرات المفتوحة مرتّبة بالأولوية. |
| `docs/partner-deck.pptx` | عرض الشريك المؤسس (١٢ شريحة). |

## التشغيل محلياً

```bash
cd diwaniya
python3 -m http.server 8000
# افتح: http://localhost:8000/index-full.html
```

اختبار المحرّك بالـ Terminal:
```bash
node -e "const {generate}=require('./engine.js'); console.log(JSON.stringify(generate('شريت سيارة جديدة وعرضتها قدام الناس'),null,2));"
```

## الانتقال للنموذج اللغوي الحقيقي
شوف `llm-bridge.js` — فيه `SYSTEM_PROMPT` + `generateWithLLM()` + نمط احتياطي (fallback).
استبدل `generate()` القواعدية بـ`generateWithLLM()` والباقي يشتغل نفسه.

**القرار الحالي (موثّق):** نبقى على المحاكي القواعدي المحسّن مؤقتاً، ونجهّز كل شي للربط بـ Claude لاحقاً بمفتاح API + Prompt Caching لتقليل التكلفة.

## للفريق التقني
- البرومبت الكامل بـ `docs/engine-spec-v4.6.md`.
- نقاط المعايرة (عتبات الخطر، التصعيد التراكبي) موسومة بالكود وقابلة للضبط.
- المحاكي يبقى كـ fallback — التجربة ما تنكسر أبداً.
- حدود المحتوى الحسّاس: «نصوّر النار، ما نشعلها» — نتوقّع ونحذّر من الإساءة، ما ننتجها.
